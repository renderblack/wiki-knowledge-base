# 虚拟技术部2026年部门说明书

## 部门定位

**业务赋能的"可视化技术中枢"**

| 属性 | 说明 |
|------|------|
| **对内** | 赋能营销、方案设计、施工管理的"可视化工具"——三维模型、动画，让业务信息更直观、决策更高效 |
| **对外** | 可视化产品与系统的"价值输出窗口"——为客户提供定制化解决方案，增强客户连接，成为新营收增长点 |

**核心能力：** 三维可视化、实景合成、动画制作、系统平台搭建，覆盖业务全链路。

---

## 2026年核心目标

### 内部业务目标
- **平台覆盖：** 依托《交通工程三维可视化平台》，完成营销中心100%在跟项目场景三维模型搭建，实现项目基础信息"一键可视化呈现"
- **业务串联：** 支撑营销、方案设计、施工策划的核心可视化需求；同步响应研发、安全等部门≥80%的临时性可视化需求

### 外部经营目标
- **产值：** 推广可视化产品+施工进度管理系统，全年实现**2000万营收**
- **客户研究：** 深度调研合作客户≥5家，挖掘2个以上客户"核心真实需求"

---

## 组织架构（4组）

> 采用"需求开发→技术落地"闭环架构

| 组别 | 负责人 | 核心职责 |
|------|--------|---------|
| 外部虚拟业务开发组 | 王沅蔓 | 对接外部客户，挖掘需求+输出方案；跟进订单落地，支撑2000万产值 |
| 内部虚拟业务开发组 | 沈晨 | 对接内部部门，收集营销/施工策划需求；推动平台模型全覆盖 |
| 视觉技术组 | 陈咪 | 实景合成、动画、平面设计；完成三维桥梁建模、地形生成 |
| 软件开发组 | 何锦平 | 开发/运维可视化平台、施工进度管理系统；攻关技术难点，保障系统稳定与创新 |

**分工逻辑：**
- 业务组与技术组分离 → 避免"需求+落地"混岗效率低下
- 内外部组拆分 → 适配"内部赋能+外部营收"双目标

---

## 关键工作路径

**内部平台覆盖路径：**
```
内部业务组收集营销项目 → 视觉组建模并同步至平台 → 审核后开放给营销使用
```

**外部产值达成路径：**
```
外部业务组对接客户 → 输出可视化方案 → 技术组完成产品制作 → 开发组完成产品封装 → 交付客户并跟进回款
```

**跨组协作机制：** 每周至少1次需求对接会，业务组同步需求、技术组同步进度
