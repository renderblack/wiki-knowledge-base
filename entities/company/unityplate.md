# UnityPlate（交通工程项目三维可视化平台）

> **内部代号**：UnityPlate = Unity（引擎）+ Platform（平台）
> **版本**：V1.0 ~ V1.3，2025年8月-12月开发，已内部验收

## 核心定位

集「**三维实景数字底座**」+「**全流程业务协同**」+「**数据资产化管理**」于一体的综合性业务操作平台。

**解决痛点**：交通工程项目「信息割裂、协同效率低、过程不透明」——数据散落各部门，无人机航测、三维模型、设计图纸、进度报告缺乏统一管理。

**聚焦阶段**：标前（项目营销 → 数据采集 → 场景解读 → 方案设计 → 成果管理），是远期覆盖施工/运维阶段的基础。

## 核心功能模块

### 模块1：多角色协同工作台
- 营销工程师、业务管理员（组/个人）、虚拟技术部（组长/组员）、场景研究室、营销中心总监办、后勤保障部、系统管理员
- 每个角色有独立定制界面

### 模块2：三维GIS可视化引擎
- 多底图：天地图、Cesium地图、线路图
- 多图层管理（拖拽、分级、飞行视角）
- 3D模型编辑（移动、旋转、分块）
- 标绘系统（点/线/面/特殊图标）
- 属性关联（桥梁类型、产品需求、工程量）
- 测量工具（距离、面积、高度）
- 飞行漫游（进度条控制）
- 场景对比（双屏联动，不同时间航测对比）
- 数据导入：KML、倾斜摄影、3DGS、图片、视频、全景图

### 模块3：企业级文件数据库
- 按项目/部门结构化存储
- 多维度搜索、在线预览、分级管理

### 模块4：无人机全流程管理
- 任务分配 + 设备匹配 + 位置监控
- 飞手白名单 + 培训考核
- 事故处理（三级审批流程）
- 设备资产全生命周期管理

## 技术栈

| 层次 | 技术选型 |
|------|---------|
| 架构 | Spring Boot + Spring MVC + MyBatis/MyBatis-Plus + Hibernate 5 |
| 数据库 | SQL Server 2012（124.71.144.65:14332） |
| 缓存 | Redis（192.168.38.151:6379） |
| 对象存储 | MinIO（S3兼容，upload.zjxtql.com） |
| 地理引擎 | Mars3D平台 |
| 权限模型 | RBAC |
| 外部集成 | 企业微信API、DAS系统（模型同步）、**A8系统（数据同步）** |

## 验收状态

| 缺陷级别 | 修复率 |
|---------|--------|
| 致命缺陷 | 100% |
| 严重缺陷 | 95% |
| 一般缺陷 | 80% |

**遗留问题（3项）**：
1. 并发上传响应延迟（交换机性能瓶颈，非软件缺陷）
2. 场景模型亮度/光照问题（Mars3D平台限制）
3. 数据过多时整体运行效果变差

## 对照集成平台落地方案

**符合**：
- ✅ 多角色工作台深入场景
- ✅ 业务即采集（航拍成果实时上传）
- ✅ 看板分层（GIS总览→各角色工作台→管理层进度条）
- ✅ 任务驱动流程（下发→流转→反馈→审核）

**差距**：
- ⚠️ 数据层薄弱：未覆盖MDM/主数据治理
- ⚠️ 集成层缺失：无事件总线、无统一主数据ID
- ⚠️ 仅覆盖标前阶段，施工/运维未打通
- ⚠️ 组织阻力：各部门原有工作习惯差异大，流程再造遭遇阻力
