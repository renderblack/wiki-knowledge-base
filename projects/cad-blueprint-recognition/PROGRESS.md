# CAD 图纸识别 — 项目进度

> 从桥梁 CAD 图纸（PDF/DWG/DXF）中自动提取结构化数据，应用于桥梁大临钢结构业务。

## 课题背景

**业务场景**：兴土股份桥梁大临钢结构业务，需处理大量设计院交付的 CAD 图纸（PDF/DWG/DXF）。

**核心目标**：从 CAD 图纸中自动提取结构化数据，实现：
1. 设计院交付图纸的自动解析入库
2. 既有图纸的数字化存档检索
3. 多项目综合分析（跨项目构件统计、算量）

## 当前阶段：Phase 0 完成 ✅ → Phase 1 待启动

### 已完成（Phase 0）

| 能力 | 验证结果 |
|------|---------|
| PDF 图纸解析 | ✅ 6页完整提取（登里1号高架特大桥） |

| 数据入库 | ✅ SQLite 4张表、101条记录 |
| 数据查询统计 | ✅ SQL 聚合查询 |
| 方向确认 | ✅ 放弃"画图"，聚焦"提取" |

### 核心缺失

| 能力 | 现状 |
|------|------|
| **构件语义识别** | 🔴 核心缺失 — 能提取几何，无法判断"桩基/墩身/梁段" |
| DWG 直接解析 | 🟡 需转 DXF 或 ODA SDK |
| PostgreSQL | 🟡 待升级 |

### 策略调整

```
调整前：CAD图纸 → 提取数据 → 画施工图 → 失败
调整后：CAD图纸 → 提取数据 → 构件识别 → 入库 → 查询/统计
```

**核心认知**：数据提取 ≠ 画图。目标是提取数据，不是替代 AutoCAD 画图。

## 构件识别优先级

| 优先级 | 构件 | 图层特征 | 几何特征 |
|--------|------|---------|---------|
| P0 | **桩基** | 桩基、钢护筒、pile | 圆形，R=300~1500mm |
| P0 | **墩身** | 墩身、pier | 封闭多段线，面积>1m² |
| P0 | **梁段** | 梁段、箱梁、T梁 | 多段线，跨度5~50m |
| P1 | 盖梁 | 盖梁、cap | 横向多段线，宽度2~5m |
| P1 | 支座垫石 | 支座、bearing | 圆形/矩形，R<300mm |
| P2 | 钢筋图 | — | — |
| P2 | 预应力钢束 | — | — |
