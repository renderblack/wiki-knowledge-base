## 一、课题概述

### 1.1 背景与目标

**背景**：兴土股份在桥梁大临钢结构业务中，需处理大量设计院交付的CAD图纸（PDF/DWG格式）。

**核心目标**：从桥梁CAD图纸中自动提取结构化数据，实现：
1. 设计院交付图纸的自动解析入库
2. 既有图纸的数字化存档检索
3. 多项目综合分析（跨项目构件统计、算量）

### 1.2 优先级构件

| 优先级 | 构件 | 说明 |
|--------|------|------|
| P0 | **桩基** | 钻孔灌注桩、钢护筒 |
| P0 | **墩身** | 桥墩主体结构 |
| P0 | **梁段** | 箱梁、T梁的分段结构 |
| P1 | 盖梁 | 墩顶横向联系梁 |
| P1 | 支座垫石 | 支座及垫石 |
| P2 | 钢筋图 | 主筋、箍筋配置 |
| P2 | 预应力钢束 | 预应力筋布置 |

### 2.3 关键发现

#### 发现1：数据提取技术可行

- DXF图纸6,356个实体全部可提取
- 579个块引用可统计
- 23个图层可分类
- 文字、尺寸、坐标均可提取

#### 发现2：构件语义识别是核心缺失

- 能提取几何数据（圆、直线、多段线坐标）
- 但无法自动判断"这是桩基"还是"这是墩身"
- 需要建立识别规则

#### 发现3：画施工图是错误方向

- EzDXF无法复制专业块定义（308实体的钢板桩块）
- 10种专业标注样式无法复现
- 施工图需要AutoCAD原版

### 4.3 构件识别规则设计

#### 识别特征表

| 构件 | 图层关键词 | 形状特征 | 尺寸范围 | 示例 |
|------|-----------|---------|---------|------|
| 桩基 | 桩基,钢护筒,pile,steel_pipe | 圆形CIRCLE | R=300~1500mm | 720锁扣桩 |
| 墩身 | 墩身,pier,pier_body | 封闭LWPOLYLINE | 面积>1m² | 3#墩 |
| 梁段 | 梁段,箱梁,T梁,girder,segment | 多段线 | 跨度5~50m | 40mT梁 |
| 盖梁 | 盖梁,cap_beam | 横向多段线 | 宽度2~5m | - |
| 支座 | 支座,bearing | 圆形/矩形 | R<300mm | 盆式支座 |

#### 识别流程

```
几何实体
    ↓
提取特征
├── 图层名称匹配
├── 形状类型判断
├── 尺寸范围筛选
└── 位置关系分析
    ↓
匹配规则库
    ↓
构件分类 + 置信度
    ↓
结构化输出
```

### 4.4 数据模型扩展

```sql
-- 构件识别结果表
CREATE TABLE identified_components (
    id SERIAL PRIMARY KEY,
    source_entity_id TEXT,      -- 原始实体ID
    component_type TEXT,         -- 构件类型
    component_name TEXT,         -- 构件名称
    confidence REAL,             -- 识别置信度 0~1
    geometry_type TEXT,          -- 几何类型
    layer_name TEXT,             -- 图层
    dimensions JSONB,            -- 尺寸参数
    coordinate JSONB,           -- 坐标
    source_drawing TEXT,         -- 来源图纸
    created_at TIMESTAMP DEFAULT NOW()
);
```
