## 核心定位

**一句话：** 集"三维实景数字底座"+"全流程业务协同"+"数据资产化管理"于一体的综合性业务操作平台。

**解决痛点：** 交通工程项目"信息割裂、协同效率低、过程不透明"——数据散落各部门，无人机航测、三维模型、设计图纸、进度报告缺乏统一管理。

**定位：** 聚焦"标前"阶段（项目营销→数据采集→场景解读→方案设计→成果管理），是远期覆盖施工/运维阶段的基础。

## 核心功能模块

### 模块1：多角色协同工作台

| 角色 | 工作台功能 |
|------|-----------|
| 营销工程师 | 项目总览、飞行任务、解读任务、成果分享、事故反馈 |
| 业务管理员（组长） | 全组项目总览、进度跟进、异常处理、解读配置 |
| 业务管理员（个人） | 所负责组内项目、任务分配、进度跟进 |
| 虚拟技术部组长 | 任务接收与二次分配、进度跟踪、成果审核 |
| 虚拟技术部组员 | 航线规划、模型制作、实景沙盘、航拍成果审核 |
| 场景研究室 | 桥梁结构分析、产品解读、信息标注 |
| 营销中心总监办 | 无人机白名单、飞行记录、设备分配、位置监控 |
| 后勤保障部 | 设备资产管理、事故记录 |
| 系统管理员 | 权限配置、日志审计、系统维护 |

### 模块2：三维GIS可视化引擎

核心能力：
- 多底图支持（天地图、Cesium地图、线路图）
- 多图层管理（图层拖拽、分级、飞行视角）
- 3D模型编辑（移动、旋转、分块管理）
- 标绘系统（点/线/面/特殊图标，颜色库）
- 属性关联（桥梁类型、产品需求、工程量等）
- 视图控制（罗盘、三视图切换）
- 测量工具（距离、面积、高度）
- 飞行漫游（相机视角规划、进度条控制）
- 场景对比（双屏联动，不同时间航测对比）
- 数据导入（KML、倾斜摄影、3DGS、图片、视频、全景图）
- 编辑权限控制（防覆盖机制）

### 模块3：企业级文件数据库

- 按项目/部门结构化存储
- 多维度搜索
- 在线预览与编辑
- 文件分级管理（下载限速、断点续传）

### 模块4：无人机全流程管理

- 任务分配 + 设备匹配
- 位置监控（地图可视化）
- 事故处理（三级审批流程）
- 飞手白名单 + 培训考核
- 设备资产全生命周期管理

## 技术方案

| 维度 | 技术选型 |
|------|---------|
| 架构 | Spring Boot + Spring MVC + MyBatis/MyBatis-Plus + Hibernate 5 |
| 数据库 | SQL Server 2012（124.71.144.65:14332，UnityPlate） |
| 缓存 | Redis（192.168.38.151:6379） |
| 对象存储 | MinIO（S3兼容，upload.zjxtql.com） |
| 地理引擎 | Mars3D平台 |
| 权限模型 | RBAC |
| 外部集成 | 企业微信API（认证+消息推送）、DAS系统（模型同步）、A8系统（数据同步） |
| 性能目标 | Web端大型场景加载≤15秒；移动端适配 |

## 验收状态

| 缺陷类型 | 修复率 |
|---------|--------|
| 致命缺陷 | 100% |
| 严重缺陷 | 95% |
| 一般缺陷 | 80% |
| 建议类 | 50% |

**尚未解决的问题（3项）：**
1. **并发上传响应延迟** — 交换机性能瓶颈，非软件缺陷
2. **场景模型亮度/光照问题** — Mars3D平台光照不符合需求
3. **数据加载缓慢** — 数据过多时整体运行效果变差

## 对照集成平台落地方案

**符合的原则：**
- ✅ 多角色工作台深入场景（每个角色定制界面）
- ✅ 业务即采集（航拍成果实时上传）
- ✅ 看板分层（GIS总览→各角色工作台→管理层进度条）
- ✅ 任务驱动流程（任务下发→流转→反馈→审核）

**差距与风险（显著）：**
- ⚠️ **数据层薄弱**：文档明确提到"数据治理"是痛点，但平台未覆盖MDM/主数据
- ⚠️ **集成层缺失**：仅对接了DAS/A8，但无事件总线、无统一主数据ID
- ⚠️ **生态闭环未完成**：仅覆盖"标前"阶段，施工/运维阶段未打通
- ⚠️ **组织阻力风险**：明确承认"各部门原有工作习惯差异大，流程再造遭遇组织阻力"
