## 核心结论（3条）

1. **最佳落点**：「高重复+高规则+高风险」的 ToB 业务环节，ROI 明确可量化
2. **核心架构**：「数据层→推理层→闭环层」三层，工业场景必须加「物理执行层」
3. **关键成功因素**：不是AI有多强，而是能不能形成「感知-决策-执行-反馈」闭环

## 四类共性痛点

```
① 数据/信息过载+分散  → 人工筛选效率极低
② 依赖个人经验       → 结果一致性差、风险不可控
③ 业务流程碎片化      → 缺乏闭环优化机制
④ 结构化输出能力弱    → 决策支撑不足
```

---

## 三层架构规律

```
数据层    ETL自动采集多源数据 → 标准化数据池
  ↓
推理层    领域知识库 + 规则/案例引擎 → 自主决策
  ↓
闭环层    用户反馈 → 知识库迭代 → 越用越精准
```

> ⚠️ 工业场景必须加第4层：「物理执行层」——兴发集团案例教训：传统AI"只思考不执行"，价值有限。

---

## 兴发集团案例（与UnityPlate最相关）

**核心矛盾**：「虚拟思考」与「物理执行」割裂

- 流程工业生产参数多（数千个实时数据），人工监控难以毫秒级响应
- 传统AI仅能提供分析建议，无法直接介入生产执行
- 生产调控依赖工程师经验，参数优化精度不足

**兴发解决思路**：构建工业生产工艺知识库 + 实时生产参数 → 「智能决策引擎+通用控制系统」→ 感知-决策-执行闭环

**对兴土的映射**：
UnityPlate 集成平台的价值方向 = 兴发的「智能决策引擎」+ 兴土设备/传感层的「物理执行」

## 跨行业痛点挖掘方法（3维度）

```
维度1  高耗时的重复劳动    → 筛选/整合/标准化
维度2  依赖个人经验        → 结果一致性差
维度3  高风险且可控性差    → 监管合规/风险预警
```

### 4步落地法

1. **场景聚焦** — 选1个核心痛点，不追求大而全
2. **知识库+数据层** — 沉淀领域经验，整合内外部数据
3. **推理引擎+POC** — 规则+案例引擎，局部验证
4. **全流程闭环** — 反馈优化，持续迭代
