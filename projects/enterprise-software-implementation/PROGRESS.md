# 传统企业软件落地

> 传统企业（尤其基建/桥梁施工行业）平台型系统落地的完整方法论与方案。

## 核心问题

传统企业里，什么样的软件/系统平台才能真正搭建并投入使用？

答案不是技术选型问题，而是**组织与人性**的问题——本质上受[[projects/differential-social-order/PROGRESS|差序格局]]支配。

## 方案目标

搭建一个能够在传统企业内真正投入使用并持续演进的平台：

- 最高管理层牵引（Top）
- 打通主营业务端到端链路（Connect）
- 深入一线场景形成可用闭环（BP）
- 数据回流持续优化（Enable + Feedback）
- 具备引入 AI/Agent 的基础（可演进）

## 8大成功要素

### 原有4点（核心成功要素）

| 要素 | 目的 | 关键机制 |
|------|------|----------|
| **Top** | 获取资源、打破认知差与跨部门阻力 | 一把手业务Owner、预算承诺、跨部门KPI协调、治理机制 |
| **Connect** | 打通端到端主链路，避免平台变孤岛 | 流程连接（端到端链路）+ 数据连接（统一主数据与口径） |
| **BP** | 平台进入岗位作业场景，围绕痛点设计 | 三件套：场景地图、最小可用闭环（MVP）、流程+规则+例外处理 |
| **Enable + Feedback** | 一线获益 + 数据回流形成飞轮 | 业务即采集、就地反馈、闭环责任 |

### 补充4个硬条件

| 条件 | 风险 | 可落地定义 |
|------|------|------------|
| **可实施** | 有任务≠自然形成组织 | 任务必须绑定：权责边界（RACI）、资源承诺（工时/预算/产能）、验收标准、例外路径 |
| **可集成** | 难点不在接口而在主数据与语义 | 统一身份/组织、主数据治理（MDM）、业务对象统一ID与事件总线、明确"主系统归属" |
| **可运营** | 把平台当产品而非项目 | 指标闭环+数据质量+需求版本+用户adoption；看板分层（一线/管理/战略） |
| **可演进** | AI成功不取决于时间而取决于数据可用性 | 以"对象+事件+反馈+审计"为底座；AI分阶段：辅助→建议→自治（有闸口/可回滚/可审计） |

## 平台总体架构（5层）

| 层次 | 内容 |
|------|------|
| 战略目标层 | OKR/经营指标/项目制；目标→里程碑→优先级→资源原则 |
| 战略执行层 | 任务分解（责任人、依赖协作、验收口径、例外处理）；交付件/验收机制 |
| 业务编排+集成层 | 流程引擎；API/事件总线；存量系统对接（ERP/财务/OA/MES/WMS/CRM）；SoR主系统归属 |
| 数据资产层 | MDM；对象字典（合同/订单/项目/工单）；事件字典；指标字典；数据质量规则 |
| AI/Agent层 | 辅助（检索/填单）→ 建议（预测/预警）→ 自治（边界内自动执行） |

**治理底座**（贯穿各层）：统一身份/权限、审计日志、合规、回滚预案

## 建设路径（3年）

| 阶段 | 时间 | 目标 | 关键交付 |
|------|------|------|----------|
| MVP闭环 | 0–6个月 | 一条主链路可跑、可 用、可回流 | 选定主链路；5–10个核心对象；打通2–3个存量系统；一线工作台上线；初版管理看板上线 |
| 扩链路+MDM | 6–18个月 | 从单链路示范到可复制 | 主数据治理机制落地；扩展2–3条链路；指标体系分层完善；运营机制稳定运行 |
| 数据资产化+AI | 18–36个月 | 可控AI落地 | 辅助型AI（检索/填单）→ 建议型AI（预测/预警）→ 自治型Agent试点 |

**核心原则**：先闭环再扩张；MVP成功标准是"可跑、可用、可回流"，不是"功能齐全"。

## 组织与治理

### 必要角色

- **业务Owner**（最高管理层授权）：跨部门拍板与资源协调
- **平台负责人**（产品化思维）：路线图、版本、价值交付
- **业务BP/流程负责人**：场景地图、岗位作业、例外规则
- **数据负责人/MDM管理员**：口径与质量治理
- **IT集成负责人**：接口、事件、权限、安全
- **超级用户网络**：各部门1–3名，负责培训、反馈、推广

### 关键制度

- 项目章程（范围、目标、资源承诺、升级机制）
- 指标字典与口径变更流程
- 需求分级与版本发布机制（S/A/B/C级）
- 数据质量例会与整改闭环

### 会议节奏

- 每周：MVP推进例会（问题清单、依赖协调、风险暴露）
- 每两周：版本评审会（需求入池、优先级、验收回顾）
- 每月：经营/指标复盘会（指标→原因→动作→负责人→截止时间）
- 每季度：战略委员会复盘（方向调整、资源重配、范围升级）

## MVP核心模板

### 对象清单（示例——按链路选）

| 对象 | 业务定义 | 主系统(SoR) | 平台角色 | 关键字段 |
|------|----------|-------------|----------|----------|
| 合同 | 客户签署的交易协议 | CRM/ERP | 编排+汇总 | 客户、金额、交付期 |
| 订单 | 内部执行凭证 | ERP | 编排 | 数量、物料、交期 |
| 项目 | 交付执行单元 | ERP/项目系统 | 编排+汇总 | 里程碑、责任人 |
| 工单 | 一线执行任务 | MES/现场系统 | 执行+反馈 | 状态、实际工时 |

### 事件清单（每对象5–15个）

| 事件类型 | 说明 |
|----------|------|
| 创建/提交 | 记录创建时间、创建人、初始字段 |
| 审批通过/驳回 | 审批人、意见、驳回原因 |
| 关键字段变更 | 金额、交付期、数量、物料等变更记录 |
| 状态流转 | 锁定、释放、完成、作废 |
| 异常事件 | 缺料、超期、质量不合格、对账差异 |

### 验收指标

| 类别 | 指标 | 口径 | 目标 |
|------|------|------|------|
| 价值 | 端到端周期缩短 | A事件到B事件的时长 | -20% |
| 价值 | 异常闭环时效 | 异常→关闭平均时长 | -30% |
| 使用 | 线上化率 | 线上处理量/总量 | ≥80% |
| 使用 | 一线活跃度 | DAU/MAU、关键功能使用次数 | 达标 |
| 数据 | 关键字段完整率 | 必填字段缺失率 | ≤2% |
| 数据 | 口径一致性 | 同指标跨系统差异率 | ≤1% |
