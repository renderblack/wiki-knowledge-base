## 一、需求背景

bolt老大需要处理已训练好的 GridMaster 3DGS 分块模型：
- **路径**：`E:\3DGS模型 新塍至王江泾524国道\`
- **规模**：137 个 Tile，L20 全量 3.69 GB（L15-L21 六级 LOD）
- **痛点**：全量加载卡顿，上云端慢，网页浏览慢
- **约束**：必须保证 L20 精度

### 学术方案调研结论

| 方案 | 问题 |
|------|------|
| 特征平面压缩（Triplane） | 需训练阶段介入，模型已训练完毕无法套用 |
| VQ/PCA 量化 | 针对隐式特征场压缩，非显式 .ply 属性 |
| 前馈压缩网络 | 依赖配套训练框架和原始训练数据 |
| Octree 空间分区 | Tile 分块本质已是空间分区，无额外收益 |
| Mipmap/LOD | 已有 LOD（L10/L15/L20），收益有限 |

**结论**：在已训练 .ply 文件前提下，唯一可行路径是**格式转换 + 数值压缩**。

## 二、数据深度分析

### 模型结构

| 指标 | 数值 |
|------|------|
| Tile 总数 | 137 个 |
| 网格范围 | 20 × 18（实际 137 个非满格分布） |
| SRS | EPSG:4549 |
| SRSOrigin | (562378.833, 3407695.236, -4.975) |
| 每高斯原始属性 | 17 个 float32 = **68 bytes** |
| L20 单 tile 大小 | ~27 MB（~74 万高斯） |

### GridMaster PLY 格式

- 17 property（x,y,z,nx,ny,nz,f_dc_0~2,rest_0~9），68B/vertex
- header 含 `comment create by GridMaster@www.daspatial.com`
- 坐标为**局部 tile 坐标**（相对 tile origin ±30m 范围）
- 法向量 (nx/ny/nz) **全为 0** → 可删除

### 精度分析

| 方案 | 精度 | 结论 |
|------|------|------|
| 世界坐标 + float16 | ±25cm | ❌ 不可接受 |
| 局部坐标 + float16 | ±0.1cm | ✅ 远超施工场景需求 |
| uint32 mm 整型 | 1mm | ✅ 更确定 |

### 压缩工具链

- **GridMaster**：需要标准 .ply 格式
- **SuperSplat (PlayCanvas)**：支持 .compressed.ply 格式
- **splat-transform** (`@playcanvas/splat-transform`）：两者互转
  - 语法：**位置参数**（不是 `-o` 标志）
  - 压缩：`splat-transform input.ply output.compressed.ply`
  - 解压：`splat-transform input.compressed.ply output.ply`

## 三、压缩方案

### splat-transform 压缩格式（.compressed.ply）

- 14 property 标准 splat，56B/vertex（无法向量）
- chunk 打包编码：11+10+11 位坐标量化
- 解码精度误差 < 1.5e-5

### 坐标变换公式（关键！）

**正确公式：`world = SRS + raw_ply_xyz`**

```
world_x = SRS_x + raw_ply_x
world_y = SRS_y + raw_ply_y
world_z = SRS_z + raw_ply_z
```

其中 SRS Origin = (562378.8332853422, 3407695.2355452203, -4.9745618242303475)

> ⚠️ `Box.Center` **不是**坐标平移量，它只是包围盒统计中心，用于定义 Tile 空间范围。DasViewer 批量加载时自动用 `SRS + tile_xyz` 变换，合并单文件时也必须用此公式。

## 七、关键教训

1. **Box.Center 不是坐标平移量**：GridMaster 的 Box.Center 是包围盒统计中心，不是 Tile 网格坐标偏移。DasViewer 批量加载时自动用 `SRS + tile_xyz` 变换，这个逻辑没有显式文档，靠对比验证才发现。

2. **合并文件数据来源**：应该直接从原始 `Data/` 目录读取 PLY 的 xyz 加上 SRS，不能用 `Data-Decoded-World/`（因为它用了错误公式）。

3. **压缩格式的选择**：splat-transform 的 .compressed.ply 是 14 属性标准 splat，无法向量，与 GridMaster 原生格式不同。压缩目的是减小体积和加快加载，GridMaster 查看仍用原始 PLY。
