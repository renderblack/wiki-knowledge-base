## 核心定位

**一句话：** 施工进度"看得见、风险早预警、数据可追溯、管理精细化"的软硬件协同平台。

**解决痛点：** 传统进度管理"不直观、数据散、风险隐、协同弱"

**产品名：兴土智瞰**，覆盖计划编制→进度跟踪→资源调配→风险预警→报告生成全流程。

---

## 核心功能模块（8大模块）

| 模块 | 核心能力 |
|------|---------|
| 项目管理监测 | 项目基本信息、进度总览、实时看板 |
| 人员管理 | 劳务人员配置、出勤统计 |
| 设备管理 | 设备台账、使用统计 |
| 工程管理 | 桥梁结构、模型关联 |
| 任务分配 | 任务派发、进度跟踪 |
| 实时日报汇总 | 当天日报自动汇总 |
| 预警机制 | 滞后预警（强控型：天数；量化型：百分比） |
| 日报填写 | 小程序端便捷填报 |

## 关键概念

| 概念 | 说明 |
|------|------|
| 强控型任务 | 含前置任务、延隔时间，严格控制执行顺序 |
| 量化型任务 | 工时驱动，无具体施工时间，设定计划持续时间 |
| 关键路线 | 耗时最长且带前置任务设定的路线 |
| 升降式展台 | 配套硬件，嵌入高清触控屏，适配86/98/110寸 |
| 装配式移动展房 | 快速搭建/拆卸的展示空间（9×6/9×9/9×12m） |
| 航测mesh/3DGS | 地图模式，无航测数据时不显示 |
| 进度偏差 | 自动计算，颜色标识（红滞后/蓝正常/绿提前） |
| 预警屏蔽 | 误报预警可手动屏蔽，保留记录供追溯 |

## 对照集成平台落地方案

**符合的原则：**
- ✅ MVP闭环：施工进度管理是独立可闭环的场景
- ✅ BP深入场景：小程序端针对劳务人员低门槛填报
- ✅ 业务即采集：日报填报在完成业务动作时自然产生
- ✅ 看板分层：首屏看板（项目经理）+ 小程序（一线）

**待验证/待加强：**
- ⚠️ 是否串联了端到端主链路（销售→交付→回款）？
- ⚠️ 主数据治理（客户/项目/合同统一ID）是否覆盖？
- ⚠️ 数据是否回流至管理层/战略层决策？
